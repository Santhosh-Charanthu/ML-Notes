1. What is a Decision Tree?

A Decision Tree is a supervised machine learning algorithm used for both classification and regression problems.

It predicts the output by repeatedly splitting the dataset into smaller groups based on the feature that best separates the data.

Interview Definition

A Decision Tree is a supervised machine learning algorithm that predicts the output by recursively splitting the data based on the feature that best separates the data into more homogeneous groups.

2. Why is it Called a Decision Tree?

It is called a Decision Tree because the model makes decisions in a tree-like structure.

It starts from the root node, asks questions, creates branches, and ends at a leaf node, which contains the final prediction.

Example

                    Age < 30?
                   /         \
                Yes           No
               /               \
      Income > 50K?          Buy = Yes
         /      \
      Yes       No
      Buy    Don't Buy

3. Components of a Decision Tree

Root Node

The first decision made by the tree.

Example:

Age < 30?

Internal Node

Intermediate decision points.

Example:

Income > 50K?

Branch

Represents the outcome of a decision.

Example:

Yes
No

Leaf Node

The final prediction.

Example:

Buy Laptop

or

Don't Buy Laptop

4. How Does a Decision Tree Work?

Suppose we have the following dataset:

Age Income Buy Laptop
22 40K No
25 80K Yes
45 90K Yes
35 35K No

The Decision Tree asks:

Which feature best separates the data?

Possible choices:

  • Age
  • Income

It selects the feature that creates the purest groups and repeats the process recursively until a stopping condition is reached.

5. How Does It Choose the Best Split?

The tree evaluates every possible split using impurity measures.

The two most common impurity measures are:

  • Gini Index
  • Entropy (Information Gain)

The best split is the one that creates the purest child nodes.

6. Gini Index

The Gini Index measures how mixed the classes are within a node.

  • Low Gini → Pure node
  • High Gini → Mixed node

Example

Node A:

Cat
Cat
Cat

Gini = Low

Node B:

Cat
Dog
Cat
Dog

Gini = High

Interview Definition

Gini Index is an impurity measure used by Decision Trees. The algorithm chooses the split with the lowest Gini impurity.

Important Note

Scikit-Learn uses Gini Index by default.

7. Entropy & Information Gain

Entropy also measures impurity.

  • Low Entropy → Pure node
  • High Entropy → Mixed node

The Decision Tree chooses the split with the highest Information Gain, which means the largest reduction in entropy.

Interview Definition

Entropy measures impurity, while Information Gain measures how much impurity is reduced after a split. The Decision Tree chooses the split with the highest Information Gain.

Gini vs Entropy

Gini Index Entropy
Measures impurity Measures impurity
Faster to compute Slightly slower
Default in Scikit-Learn Used with criterion="entropy"

8. Decision Tree for Classification & Regression

Classification

Predicts categories.

Examples:

  • Spam / Not Spam
  • Pass / Fail
  • Disease / No Disease

Regression

Predicts continuous values.

Examples:

  • House Price
  • Salary
  • Temperature

In regression, each leaf node stores a numerical prediction instead of a class.

9. Overfitting

Overfitting occurs when the Decision Tree becomes too complex and memorizes the training data instead of learning general patterns.

Symptoms

  • Very high training accuracy.
  • Low testing accuracy.

Example

A very deep tree may create separate branches for a few unusual data points, reducing its ability to generalize to new data.

10. How to Reduce Overfitting

Common techniques include:

1. Limit Tree Depth

max_depth=3

This prevents the tree from becoming too deep.

2. Increase Minimum Samples Required to Split

min_samples_split=5

This prevents unnecessary splits.

3. Increase Minimum Samples per Leaf

min_samples_leaf=2

This ensures each leaf contains enough data.

4. Pruning

Remove branches that contribute little to prediction accuracy.

11. Advantages

  • Easy to understand and interpret.
  • Easy to visualize.
  • Handles both numerical and categorical data.
  • No feature scaling required.
  • Can solve both classification and regression problems.
  • Captures non-linear relationships.

12. Limitations

  • Can easily overfit.
  • Sensitive to small changes in the dataset.
  • Deep trees become difficult to interpret.
  • Usually less accurate than ensemble methods like Random Forest.

Why Doesn't a Decision Tree Need Feature Scaling?

Decision Trees make decisions using thresholds.

Example:

Age < 30?

Salary > 50000?

The actual scale of the feature doesn't affect these comparisons.

Interview Answer

Decision Trees do not require feature scaling because they split data based on feature thresholds. Scaling changes the values but preserves their order, so the split decisions remain the same.

Scikit-Learn Implementation

from sklearn.tree import DecisionTreeClassifier
from sklearn.datasets import load_iris

iris = load_iris()

X = iris.data
y = iris.target

model = DecisionTreeClassifier(
    criterion="gini",
    max_depth=3,
    random_state=42
)

model.fit(X, y)

prediction = model.predict([X[0]])

print("Prediction:", prediction)

Common Parameters

Parameter Purpose
criterion "gini" or "entropy"
max_depth Maximum depth of the tree
min_samples_split Minimum samples required to split a node
min_samples_leaf Minimum samples required in a leaf
random_state Ensures reproducible results

🎤 Top 10 Interview Questions with Answers

1. What is a Decision Tree?

Answer:

A Decision Tree is a supervised machine learning algorithm used for classification and regression. It predicts the output by recursively splitting the data based on the feature that best separates the data into more homogeneous groups.

2. Why is it called a Decision Tree?

Answer:

It is called a Decision Tree because it makes decisions in a tree-like structure, starting from a root node, following branches based on feature values, and ending at a leaf node containing the final prediction.

3. How does a Decision Tree choose the best feature to split?

Answer:

A Decision Tree evaluates possible splits using impurity measures such as Gini Index or Entropy. It selects the split that creates the purest child nodes by minimizing impurity or maximizing Information Gain.

4. What is the Gini Index?

Answer:

The Gini Index is an impurity measure used to evaluate how mixed the classes are in a node. Lower Gini values indicate purer nodes. Scikit-Learn uses Gini as the default criterion.

5. What is Entropy?

Answer:

Entropy measures the impurity or randomness in a node. The Decision Tree selects the split with the highest Information Gain, which corresponds to the greatest reduction in entropy.

6. What is Information Gain?

Answer:

Information Gain measures how much a split reduces entropy. A higher Information Gain indicates a better split because it creates purer child nodes.

7. What is overfitting in a Decision Tree?

Answer:

Overfitting occurs when the tree becomes too deep and memorizes the training data instead of learning general patterns, resulting in poor performance on unseen data.

8. How can overfitting be reduced?

Answer:

Overfitting can be reduced by limiting the tree depth (max_depth), increasing min_samples_split, increasing min_samples_leaf, or pruning unnecessary branches.

9. Why doesn't a Decision Tree require feature scaling?

Answer:

Decision Trees split data based on thresholds, such as Age < 30. Since scaling preserves the order of values, it does not affect the split decisions, so feature scaling is unnecessary.

10. What are the advantages and limitations of Decision Trees?

Answer:

Advantages:

  • Easy to understand and visualize.
  • Handles numerical and categorical data.
  • No feature scaling required.
  • Supports both classification and regression.
  • Captures non-linear relationships.

Limitations:

  • Prone to overfitting.
  • Sensitive to small changes in data.
  • Deep trees become complex.
  • Usually less accurate than ensemble methods like Random Forest.
