1. What is Random Forest?

Random Forest is an ensemble learning algorithm used for both classification and regression.

Instead of relying on a single Decision Tree, it builds multiple Decision Trees and combines their predictions to produce the final output.

Interview Definition

Random Forest is an ensemble learning algorithm that builds multiple Decision Trees using different subsets of the training data and combines their predictions using majority voting (classification) or averaging (regression). This improves accuracy and reduces overfitting.

2. Why Do We Need Random Forest?

A single Decision Tree can easily overfit the training data.

Example:

Training Accuracy = 100%
Testing Accuracy  = 78%

This happens because the tree memorizes the training data instead of learning general patterns.

Random Forest solves this problem by combining predictions from many Decision Trees, making the final model more robust.

3. Why is it Called Random Forest?

It is called Random Forest because randomness is introduced in two ways:

1. Random Data

Each Decision Tree is trained on a different bootstrap sample of the training dataset.

2. Random Features

At every split, each Decision Tree considers only a random subset of features instead of all available features.

These two sources of randomness create diverse trees and improve generalization.

4. Bootstrap Sampling

Bootstrap Sampling is the process of creating multiple training datasets by sampling with replacement from the original dataset.

Example

Original dataset:

A
B
C
D
E

Possible bootstrap samples:

Tree 1

A
B
B
D
E

Tree 2

A
C
C
D
E

Notice:

  • Some samples appear multiple times.
  • Some samples may not appear at all.

Interview Definition

Bootstrap Sampling is the process of creating multiple training datasets by randomly sampling the original dataset with replacement.

What Does "Sampling with Replacement" Mean?

Suppose you have five balls:

A B C D E

You pick one ball.

Suppose you pick:

B

You put it back into the bag before picking again.

Now B can be selected again.

This is called sampling with replacement.

5. Bagging (Bootstrap Aggregating)

Bagging stands for Bootstrap Aggregating.

It consists of three steps:

Step 1

Create multiple bootstrap datasets.

↓

Step 2

Train one Decision Tree on each dataset.

↓

Step 3

Combine all predictions.

Classification Example

Tree 1 → Cat
Tree 2 → Dog
Tree 3 → Cat
Tree 4 → Cat
Tree 5 → Dog

Final Prediction:

Cat

using Majority Voting.

6. Random Feature Selection

Suppose the dataset contains:

  • Age
  • Salary
  • Experience
  • Education
  • City

A Decision Tree considers all features while finding the best split.

Random Forest considers only a random subset of features for each split.

Example:

Tree 1:

Age
Salary

Tree 2:

Experience
Education

Tree 3:

Salary
City

This increases diversity among the trees and reduces correlation.

7. How Random Forest Works

Step 1

Create multiple bootstrap samples.

↓

Step 2

Train one Decision Tree on each sample.

↓

Step 3

Each tree makes its own prediction.

↓

Step 4

Combine predictions.

  • Classification: Majority Voting.
  • Regression: Average of predictions.

8. Classification vs Regression

Classification

Each tree predicts a class.

Example:

Tree 1 → Spam
Tree 2 → Spam
Tree 3 → Not Spam
Tree 4 → Spam
Tree 5 → Spam

Final Prediction:

Spam

using majority voting.

Regression

Each tree predicts a numerical value.

Example:

Tree 1 → ₹22 Lakh
Tree 2 → ₹24 Lakh
Tree 3 → ₹23 Lakh
Tree 4 → ₹25 Lakh

Final Prediction:

Average

= (22 + 24 + 23 + 25) / 4

= ₹23.5 Lakh

9. How Random Forest Reduces Overfitting

A single Decision Tree has high variance, meaning it is sensitive to small changes in the training data.

Random Forest reduces overfitting because:

  • Each tree is trained on a different bootstrap sample.
  • Each tree considers a random subset of features.
  • The final prediction combines the outputs of many trees.

Individual mistakes made by one tree are often corrected by the others.

Interview Answer

Random Forest reduces overfitting by training multiple Decision Trees on different bootstrap samples and random subsets of features. The final prediction combines all trees, reducing variance and improving generalization.

10. Feature Importance

Random Forest can estimate the importance of each feature.

Example:

Feature Importance
Experience 0.42
Salary 0.31
Education 0.18
Age 0.09

A higher value indicates that the feature contributes more to the model's predictions.

11. Advantages

  • Higher accuracy than a single Decision Tree.
  • Less prone to overfitting.
  • Works for both classification and regression.
  • Handles missing values reasonably well.
  • Provides feature importance.
  • No feature scaling required.
  • Performs well on large datasets.

12. Limitations

  • Slower than a single Decision Tree.
  • Uses more memory.
  • Harder to interpret because many trees are involved.
  • Increasing the number of trees increases computation time.

13. Scikit-Learn Implementation

from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_iris

iris = load_iris()

X = iris.data
y = iris.target

model = RandomForestClassifier(
    n_estimators=100,
    criterion="gini",
    max_depth=5,
    random_state=42
)

model.fit(X, y)

prediction = model.predict([X[0]])

print("Prediction:", prediction)
print("Feature Importance:", model.feature_importances_)

14. Common Parameters

Parameter Purpose
n_estimators Number of Decision Trees in the forest
criterion Splitting criterion ("gini" or "entropy")
max_depth Maximum depth of each tree
min_samples_split Minimum samples required to split a node
min_samples_leaf Minimum samples required in a leaf node
random_state Ensures reproducible results

Common Interview Question

Will increasing n_estimators always improve accuracy?

Answer:

No.

Increasing n_estimators generally improves model stability and may improve accuracy initially. However, after a certain point, additional trees provide very little improvement while increasing training time, prediction time, and memory usage.

16. Top 10 Interview Questions with Answers

1. What is Random Forest?

Answer:

Random Forest is an ensemble learning algorithm that builds multiple Decision Trees on different bootstrap samples and combines their predictions using majority voting (classification) or averaging (regression) to improve accuracy and reduce overfitting.

2. Why is Random Forest better than a Decision Tree?

Answer:

Random Forest uses multiple Decision Trees instead of one. By averaging their predictions, it reduces variance, improves generalization, and is less prone to overfitting.

3. What is Bootstrap Sampling?

Answer:

Bootstrap Sampling is the process of creating multiple training datasets by randomly sampling the original dataset with replacement, allowing duplicate samples to appear.

4. What is Bagging?

Answer:

Bagging (Bootstrap Aggregating) is an ensemble technique where multiple bootstrap datasets are created, separate models are trained on each dataset, and their predictions are combined to produce the final prediction.

5. Why is it called Random Forest?

Answer:

It is called Random Forest because randomness is introduced in two ways:

  1. Random bootstrap samples of the training data.
  2. Random subsets of features considered at each split.

6. How does Random Forest reduce overfitting?

Answer:

Random Forest reduces overfitting by training multiple Decision Trees on different bootstrap samples and random subsets of features, then combining their predictions. This reduces variance and improves generalization.

7. How does Random Forest perform classification?

Answer:

Each Decision Tree predicts a class label, and the final prediction is determined using majority voting.

8. How does Random Forest perform regression?

Answer:

Each Decision Tree predicts a numerical value, and the final prediction is the average of all predictions.

Answer:

Feature Importance measures how much each feature contributes to the model's predictions. Features with higher importance have a greater influence on the final decision.

Limitations:
